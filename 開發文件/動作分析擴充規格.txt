# 動作偵測分析系統擴充開發規格

## 1. 擴充目標

系統需支援：

1. 不同拍攝角度：正面、側面、斜前方、斜後方。
2. 不同動作類型：深蹲、弓箭步、硬舉及後續新增動作。
3. 依動作與角度選擇不同關節、角度及判斷規則。
4. 輸出骨架影片、關節軌跡、動作分期、問題與改善建議。
5. 未來可替換 MediaPipe 為 ONNX，而不影響規則與前端。

---

## 2. 建議功能架構

```text
影片上傳
  ↓
影片品質檢查
  ↓
Pose Detection
  ↓
節點可信度過濾與平滑
  ↓
拍攝角度辨識
  ↓
動作類型選擇或辨識
  ↓
動作分期
  ↓
動作規則分析
  ↓
分數與問題摘要
  ↓
標註影片及分析報告
```

---

## 3. 核心模組

### 3.1 Pose Detection 模組

負責：

* 擷取 33 個人體節點。
* 保留 x、y、z、visibility。
* 支援可調整的分析幀率。
* 對節點資料進行平滑處理。
* visibility 過低時標記為無效資料。

建議介面：

```python
class PoseEstimator:
    def analyze_frame(self, frame) -> PoseFrame:
        pass
```

未來 MediaPipe、ONNX 均實作相同介面。

---

### 3.2 拍攝角度模組

系統初期不建議完全自動辨識，先由使用者選擇：

```text
front
side_left
side_right
diagonal
```

後續再增加自動判斷，依據：

* 左右肩膀水平距離。
* 左右髖水平距離。
* 左右肢體遮擋程度。
* 關節 visibility 差異。

輸出：

```json
{
  "cameraView": "side_left",
  "confidence": 0.91
}
```

---

### 3.3 關節特徵計算模組

將原本固定的四個角度，改為通用特徵計算服務。

應支援：

* 膝關節角度。
* 髖關節角度。
* 踝關節角度。
* 軀幹前傾角度。
* 骨盆傾斜。
* 肩膀傾斜。
* 左右對稱差。
* 關節移動速度。
* 關節移動軌跡。
* 身體重心近似位置。

```python
class FeatureCalculator:
    def calculate(self, pose_frame, camera_view) -> FrameFeatures:
        pass
```

不同拍攝角度只使用可信特徵，例如：

| 拍攝角度 | 適合判斷           |
| ---- | -------------- |
| 正面   | 左右對稱、膝內夾、骨盆偏移  |
| 側面   | 深度、軀幹前傾、髖膝踝角度  |
| 斜角   | 輔助觀察，但判斷門檻需較寬鬆 |

---

### 3.4 動作分析器模組

每種動作建立獨立分析器，不將規則全部寫在同一個檔案。

```python
class MovementAnalyzer:
    def analyze(self, frames, camera_view) -> MovementResult:
        pass
```

實作範例：

```text
SquatAnalyzer
LungeAnalyzer
DeadliftAnalyzer
```

新增動作時，只需新增分析器與設定，不修改 Pose Detection。

---

### 3.5 動作分期模組

分析前需先找出完整動作週期。

深蹲分期範例：

```text
站立起始
下降階段
最低點
上升階段
回到站立
```

判斷依據：

* 髖部垂直位移。
* 膝關節角度變化。
* 關節移動方向。
* 速度由負轉正的轉折點。

輸出：

```json
{
  "repetitions": [
    {
      "startFrame": 10,
      "bottomFrame": 62,
      "endFrame": 118
    }
  ]
}
```

---

### 3.6 規則引擎

規則不得寫死在 Controller 或 API Endpoint。

建議規則格式：

```json
{
  "movement": "squat",
  "cameraView": "side",
  "rules": [
    {
      "code": "SQUAT_DEPTH",
      "feature": "minKneeAngle",
      "operator": "<=",
      "threshold": 100,
      "severity": "warning"
    }
  ]
}
```

規則分為：

1. 必要條件：節點可見度、完整動作週期。
2. 動作品質：深度、穩定度、對稱性。
3. 風險提示：膝內夾、軀幹過度前傾等。
4. 資料不足：遮擋、出畫、角度不適合。

門檻值應可設定，不應散落在程式碼中。

---

## 4. 不同角度的分析策略

同一個動作需有不同角度規則。

### 深蹲側面

主要分析：

* 膝關節最小角度。
* 髖部下降幅度。
* 軀幹前傾。
* 髖膝踝伸展順序。
* 腳跟是否明顯抬起。

### 深蹲正面

主要分析：

* 左右膝角度差。
* 膝蓋是否向內偏移。
* 骨盆左右傾斜。
* 身體重心左右偏移。
* 左右腳穩定性。

不應使用正面影片精確判斷軀幹前傾，也不應只用側面影片判斷左右膝內夾。

---

## 5. 建議 API

### 分析影片

```http
POST /api/v1/motion/analyze
```

參數：

```text
file
movementType
cameraView
frameInterval
generateAnnotatedVideo
```

### 回傳格式

```json
{
  "success": true,
  "analysisId": "uuid",
  "movementType": "squat",
  "cameraView": "side_left",
  "quality": {
    "usable": true,
    "score": 86,
    "warnings": []
  },
  "repetitions": [],
  "summary": {
    "score": 82,
    "strengths": [],
    "issues": []
  },
  "outputVideoUrl": null
}
```

原始逐幀節點資料不建議全部直接回傳前端，可另設詳細資料 API，避免 JSON 過大。

---

## 6. 視覺化輸出

每次分析至少產生：

1. 人體骨架疊圖影片。
2. 關鍵關節軌跡。
3. 動作最低點或關鍵幀標記。
4. 異常位置文字標記。
5. 原始影片與分析影片並排播放。

軌跡應支援指定關節，例如：

```text
LEFT_KNEE
RIGHT_KNEE
LEFT_HIP
RIGHT_HIP
```

```text
MotionAnalysis.Web
├─ Controllers
├─ Views
├─ Application
│  ├─ MotionAnalysisService
```

* 特徵計算。
* 動作分期。
* 規則分析。
* 標註影片產生。

## 8. 開發順序

### 第二階段

1. 增加骨架疊圖影片。
2. 增加節點平滑與 visibility 過濾。
3. 增加拍攝角度選擇。
4. 建立通用關節特徵計算模組。
5. 完成側面深蹲分析。

### 第三階段

1. 增加正面深蹲分析。
2. 增加動作分期與次數計算。
3. 建立規則設定檔。
4. 增加分析分數及問題摘要。
5. 增加分析結果頁面。

### 第四階段

1. 新增弓箭步、硬舉分析器。
2. 增加拍攝角度自動判斷。
3. 增加歷史紀錄與資料庫。
4. 串接 LLM 產生自然語言回饋。
5. 評估替換為 ONNX Runtime。

---

## 9. 驗收標準

* 使用者可指定動作類型與拍攝角度。
* 系統能判斷影片是否適合分析。
* 系統能辨識至少一個完整動作週期。
* 側面與正面使用不同規則。
* 輸出骨架標註影片。
* 輸出關鍵角度、最低點、左右差異。
* 資料不足時不得強行評分。
* 新增動作時不需修改 Pose Detection 核心程式。
* 未來替換 ONNX 時，API 回傳格式保持相容。

## 10. 第一個建議實作範圍

下一版先完成：

```text
側面深蹲影片
→ 節點平滑
→ 動作分期
→ 找出最低點
→ 計算膝角、髖角、軀幹角度
→ 輸出骨架影片
→ 顯示三項主要分析結果
```

暫時不要同時開發多種動作與自動角度辨識，先以「側面深蹲」建立完整且可驗證的標準分析流程，再複製到其他動作。
